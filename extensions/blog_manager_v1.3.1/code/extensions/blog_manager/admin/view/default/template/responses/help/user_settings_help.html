User Settings

Your blog can be open to anyone to post comments or you can require your users to log in before they post a comment. If you choose to restrict access, you have the choice to use your existing and future customers as a data source or to keep access to your blog completely separate. Regardless of how you operate your blog, you still have the ability to moderate comments (see Comments tab).

Using Customer Information

When using customer information as your data source, some basic information is copied to the blog user table where additional data is kept for your blog user. When new users register, they will use your store registration process and automatically added a blog user. You can still manually add users outside of your customer base who will only have access to your blog.

If you are using your store domain for your blog, and not a different domain, store users who log into your blog will also be logged into the store and vice versa. If you are using a different domain, store users may have to log into the blog separate from logging into the store but use the same credentials.

Using Separate Login Credentials

When using separate login credentials, users will not be logged into your store when logging into the blog. When users register, they use a separate process than your store users. You have the same options for approving and activating your blog users as with your store.

When using separate login credentials, a new customer group is created: Blog Users. This will allow you to promote specials and discounts only for your blog users.

Switching Between Methods

While it is best to set up your blog using one method or another, it is possible to switch methods.

  • Switching between an open model to a login model.
    • Dependant on your login model:
      • Customers now have access with the store login credentials.
      • Existing users will have to register before posting comments.
    • All past comment history remains as is.
    • You will need to create a new user entity for each your authors and then recreate your authors with the new user id. Any articles will have to be manually moved to the new author id.
  • Switching between a login model to an open model.
    • No ability to restrict access. However, you still have ability to moderate comments
    • All user records are retained.
    • All user references in blog comments remain the same.
    • Existing users will no longer have the ability to edit their settings and notifications.
    • Admin access to users the user interface is disabled.
  • Switching between customer access and separate access.
    • All past comment history is retained.
    • Customers will no longer have access to your blog.
    • Existing manually entered users are retained.
